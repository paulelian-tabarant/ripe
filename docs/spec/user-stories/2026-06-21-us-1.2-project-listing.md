# US-1.2 — Project Listing

**Slice**: 2 — Dashboard Bootstrap  
**Status**: Done

## User Story

As a user, I can open the dashboard and see a list of registered projects, select one, and have the
rest of the dashboard reflect that selection.

## Context

This is the first visible feature in the dashboard. It replaces the walking skeleton's placeholder
with real data. The selected `project_id` drives the activity view built in US-3.1.

Frontend architecture (ADR-011): Pages / atomic components / hooks / services.  
Data fetching (ADR-008): fetch on change, no client-side cache.

## Scope

**In scope:**

- `GET /api/projects` server endpoint
- Project selector UI component
- Selected project held in React state (no browser persistence — resets on refresh)

**Out of scope:**

- Project deletion or editing
- Filtering or searching the project list
- The activity view (built in US-3.1 — the selector just exposes the selected `project_id`)

## API Contract

```text
GET /api/projects
→ 200 Array<{ id: string, name: string }>
```

## Frontend Behavior

- On load: fetches project list from `GET /api/projects`
- Renders a project selector (dropdown), initially showing a placeholder ("Select a
  project…") as the selected value — no project is auto-selected on load
- Selected project held in React state; passed down to the activity section as `project_id`
- No activity content shown until a project is selected
- Shows loading state while fetching
- Shows error state if the request fails
- Shows empty state if no projects are registered

## Acceptance Criteria

- Given the API hasn't responded yet, when `DashboardPage` renders, then a loading
  indicator is shown.
- Given `GET /api/projects` resolves with two projects, when `DashboardPage` renders,
  then both project names appear as dropdown options, and a placeholder ("Select a
  project…") is shown as the selected value.
- Given two projects are loaded, when the user selects one via the dropdown, then that
  project becomes the selected value.
- Given `GET /api/projects` resolves `[]`, when `DashboardPage` renders, then an
  empty-state message is shown and no dropdown renders.
- Given `GET /api/projects` fails, when `DashboardPage` renders, then an error message
  is shown.
